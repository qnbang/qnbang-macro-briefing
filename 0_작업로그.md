# 매크로 투자 브리핑 서비스 — 작업로그

> 거시 흐름을 먼저 읽고 "이 자산을 이런 관점으로 보라"를 기사 형식으로 자동 발행하는 서비스.
> 포지셔닝: 매수/매도 신호가 아닌 **거시 흐름 학습 + 시나리오 관찰 도구**.
> 차별화: 글로벌 거시 흐름을 **한국 투자자의 자산 언어(원화·코스피·수출산업)로 번역**.

## 2026-06-11 — MVP 개발 완성 + 배포 (라이브 + 매일 자동) ★
- **라이브 발행:** https://qnbang-macro-briefing.vercel.app (Vercel·정적 서빙)
- **저장소:** github.com/chalrie92/qnbang-macro-briefing (비공개) — GitHub→Vercel 자동배포 연결됨.
- **파이프라인 코드(브리핑기/):**
  - `fetch_market.py` — yfinance로 시세 10종 수집(코스피·코스닥·S&P500·나스닥·VIX·원/달러·달러지수·WTI·금·미10년물), 값·시점·출처 동반 → market_data.
  - `store_briefing.py` — 브리핑 본문·시나리오 이력 저장 헬퍼(이력_로그 1일차부터 축적, 신뢰도/상태 자동 반영).
  - `오늘브리핑_seed.py` — 본문 dict **형식 견본** + 첫 데일리 1편.
  - `build_site.py` — DB→정적 대시보드(시안 v2 그대로, 한국 증시 색관례). index·archive·data.json.
- **숫자/서사 분리 구현:** 지표 카드는 kind="market"(market_data 자동 인용) / kind="cited"(Tier1 출처 명시). LLM은 새 숫자 못 만듦.
- **첫 브리핑(데일리):** 유가發 인플레→연준 동결→강달러·원화 사상약세(1,531원)→외국인 이탈→코스피 -4.11%. 시나리오 5종 강화/약화 판정·로그 기록(유가인플레·고환율 강화, 연준인하·금 약화).
- **구동 결정 실현:** 토큰 API 결제 아님 → **클라우드 예약 에이전트(클로드 구독 헤드리스)**. 맥 꺼져도 매일 07:30 KST(cron `30 22 * * *` UTC) 자동: 수집→리서치→생성→렌더→git push→자동배포. 루틴 `trig_017mao9jpbxkg4EwViUXNJWk`(모델 sonnet-4-6, 추후 Opus 승격 가능). 관리: claude.ai/code/routines

## 2026-06-11 — 기획 검토·보강 (v0.1 → v0.2)
- 사용자가 v0.1 기획서 제출 → 비판적 검토·보강 진행.
- 핵심 보강 6가지:
  1. **숫자/서사 분리** — 거시 수치는 구조화 데이터 API에서만(할루시네이션 1순위 차단), 구조화 데이터를 MVP 핵심으로 승격.
  2. **3중 검증 게이트** — 수집 화이트리스트 → 제약 생성(숫자 추가 금지) → 검수 패스(팩트 대조).
  3. **시나리오 DB 스키마 확정** — 트리거/반증/파급체인/상태/이력로그. 적중 추적을 1일차부터 데이터로 축적.
  4. **모델·비용 배분** — 일간 경량(Haiku/Sonnet), 월간 이상 Opus. 비용 가드레일.
  5. **포지셔닝 명확화** — 학습·관찰 도구(유사투자자문 규제 방어선 겸 정체성).
  6. **MVP 순서 재조정** — 시나리오 DB → 구조화 데이터 → 이벤트 → 분석체인 → 기사+검수.
- 산출물: [2_기획/기획서-v0.2.md](2_기획/기획서-v0.2.md)
- **구동·비용 결정**: 토큰당 API 결제 대신 **이미 내는 클로드 구독 + 클로드 코드 자동 실행(headless)**으로 → 추가 비용 0원, Opus 품질 유지. 비용 상한 항목 폐기.
- 다음 결정 대기: 데이터 소스(FRED/ECOS 키), 시나리오 시드 주제, 코딩 착수 단계.
