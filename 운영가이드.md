# 매일 발행 운영 가이드 (클라우드 예약 에이전트용)

> 매일 아침 이 절차대로 **오늘치 데일리 브리핑 1편**을 만들어 발행한다.
> 원칙: **모든 숫자는 검증 경로에서만**. 시세는 `fetch_market.py`가 받은 `market_data`에서,
> 그 외 인용 수치(CPI·정책금리 등)는 **Tier1 출처(중앙은행·통계청·주요언론)**를 본문에 명시.
> LLM은 산문만 쓰고 **새 숫자를 만들지 않는다.** 의심스러운 수치는 **본문에서 단정하지 않는다.**

## 0. 준비 (한 번)
```bash
cd 브리핑기
python3 -m venv .venv 2>/dev/null || true
.venv/bin/pip install -q -r requirements.txt
```

## 1. 시세 수집 (숫자 경로)
```bash
cd 브리핑기 && .venv/bin/python fetch_market.py
```
→ 코스피·코스닥·S&P500·나스닥·VIX·원/달러·달러지수·WTI·금·미10년물이 `market_data`에 적재된다.

## 2. 거시 이벤트 리서치 (서사 경로)
다음 5개 축으로 웹 검색(WebSearch)해 **오늘**의 사건·맥락을 파악한다(숫자 인용은 Tier1만):
- 통화정책: 연준/한국은행 발언·CPI·금리
- 지정학·원자재: 중동·유가·호르무즈
- 환율·수급: 원/달러, 외국인 순매수/순매도
- 산업: 반도체(HBM)·수출
- 시장 총평: 코스피/글로벌 증시 그날 움직임

## 3. 브리핑 본문 작성 → DB 기록
`브리핑기/오늘브리핑_seed.py`의 **dict 구조를 그대로** 채워(어제 대비 '무엇이 바뀌었나'를 리드로),
아래처럼 저장한다. 시세 카드는 `kind:"market"`(값은 빌더가 자동 인용), 인용 수치는 `kind:"cited"`+`source`.
```python
from store_briefing import upsert_briefing, log_scenario
bid = upsert_briefing("YYYY-MM-DD", "daily", briefing_dict)
# 오늘 이벤트가 건드린 시나리오를 대조해 이력_로그 축적(강화/약화/유지)
log_scenario("scn-...", "YYYY-MM-DD", "강화", "근거 한 줄", bid, source="브리핑 분석체인")
```
시나리오 대조: 각 시나리오의 `trigger_conditions`/`counter_conditions`를 오늘 이벤트와 대조해
**강화/약화/유지**를 판정하고 근거와 함께 `log_scenario`로 남긴다. (적중 추적의 원천)

## 4. 검수 게이트
작성한 본문의 **모든 수치**가 ① `market_data`에 있거나 ② 본문 `source`에 Tier1 출처가 달렸는지 확인.
근거 없는 수치는 제거하거나 "미확인"으로 둔다.
