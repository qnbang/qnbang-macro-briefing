# 매크로 투자 브리핑 서비스 — 기획서 (v0.2)

> v0.1(사용자 제출본)을 비판적으로 검토하고 빈틈을 채운 보강판.
> 변경의 핵심: ① 숫자와 서사를 아키텍처 차원에서 분리, ② 검증 파이프라인을 3중 게이트로 구체화,
> ③ 시나리오 DB 스키마 확정(적중 추적을 1일차부터 설계에 포함), ④ 비용·모델 배분 구체화,
> ⑤ "한국 자산으로의 번역"을 차별화 축으로 명시, ⑥ MVP 순서 재조정(시나리오 DB·구조화 데이터를 먼저).

## 0. v0.1 검토 요약 — 무엇을 바꿨고 왜인가

| # | v0.1의 지점 | 문제 / 리스크 | v0.2의 결정 |
|---|------------|--------------|------------|
| 1 | 시장 데이터 API가 "(선택) 2차 확장" | 거시 **숫자를 web_search 서사·LLM에서** 뽑으면 할루시네이션 1순위 위험 | 구조화 데이터 수집을 **MVP 핵심**으로 승격. 숫자=구조화 API, 서사·해석=검색·LLM으로 **분리** |

| 4 | 적중 추적이 v0.4 | 나중에 붙이려면 과거 데이터가 없어 불가능 | **스키마에 이력로그를 1일차부터** 박아둠. 기능은 나중이라도 데이터는 처음부터 쌓음 |
| 5 | 교육 vs 실전 가치가 병렬 | 포지셔닝이 흐리면 설계·면책이 흔들림 | **"거시 흐름 학습 + 시나리오 관찰 도구"**로 명시(투자 신호 아님). 실전=후크, 교육=리텐션 |
| 6 | 글로벌 매크로 일반론 | 해외 매크로 뉴스레터와 차별화 약함 | **"이 흐름이 한국 자산(원화·코스피·수출산업)에 무슨 의미인지" 번역**을 차별축으로 |
| 7 | MVP가 일간 브리핑부터 | 기사 생성이 참조할 시나리오 DB가 없으면 공허 | MVP 순서 재조정: **시나리오 DB → 구조화 데이터 → 이벤트 → 분석체인 → 기사+검수** |

## 1. 핵심 컨셉 (유지 + 정교화)

- **방향**: 사용자 입력 → 분석(X) / 서비스가 흐름 감지 → 푸시(O) — *유지*
- **포지셔닝(명확화)**: 매수/매도 신호가 아니라 **"거시 흐름을 읽는 눈을 길러주는 시나리오 관찰 도구"**.
  실전 참고는 **후크**, 매일 읽으며 경제 보는 눈이 길러지는 것이 **리텐션 엔진**.
- **차별화 한 줄**: *"글로벌 거시 흐름을, 한국 투자자의 자산 언어로 번역한다."*
  (해외 매크로 뉴스레터는 많지만, "그래서 원달러·코스피·반도체 수출에 무슨 의미인지"까지 좁혀주는 곳은 드뭄)
- **신뢰의 핵심**: 시나리오 **적중/오류를 모두 공개**하는 정직성. "맞췄다"뿐 아니라 "이 시나리오는 빗나갔다"까지
  이력으로 남기는 것이 장기 신뢰의 엔진(= 시나리오 성적표).

## 2. 아키텍처 — 가장 중요한 원칙: **숫자와 서사의 분리**

```
[숫자 경로]  구조화 데이터 API ──┐
 (검증된 수치)  FRED/ECOS/지수   │
                                 ├──→ [분석 체인] ──→ [제약 생성] ──→ [검수 게이트] ──→ [발행]
[서사 경로]  웹검색 + RSS ───────┘     (LLM)          (LLM, 숫자       (LLM, 별도
 (이벤트·맥락)  tier 화이트리스트                       추가 금지)        팩트 대조)
```

핵심 규칙:
- **모든 숫자는 숫자 경로에서만 온다.** LLM·검색 서사에서 나온 수치는 기사에 들어갈 수 없다.
- **LLM은 산문을 쓰되 새 숫자를 만들 수 없다.** 생성 단계는 "팩트 목록"에 있는 값만 인용한다(제약 생성).
- **검수 게이트**가 생성된 기사의 모든 수치·사실 주장을 추출해 출처 목록과 대조, 미매칭은 제거/플래그.

### 2.1 수집 레이어

**숫자 경로 (MVP 핵심 — 5~10개 시리즈로 시작)**
- 금리: 미 10년물·2년물, 한국 국고 3년 (FRED, 한국은행 ECOS)
- 환율: DXY(달러지수), 원/달러 (FRED, ECOS)
- 원자재: WTI, 금 (FRED / Alpha Vantage / yfinance)
- 지수·변동성: S&P500, 코스피, VIX (yfinance)
- 각 값은 `{지표, 값, 단위, 시점, 출처, 전일대비}` 형태로 저장. **시점·출처 없는 값은 저장 금지.**

**서사 경로 (이벤트·맥락)**
- 1차(MVP): Claude `web_search`로 카테고리별 쿼리 세트(통화정책/지정학/산업·반도체/수급/원자재)
- 2차(확장): RSS(한경·매경·연합·Bloomberg·Reuters)
- **출처 tier 화이트리스트**(아래 6장)를 통과한 소스만 파이프라인에 진입. SNS·블로그·익명 글은 **차단 또는 "미확인" 라벨**.

### 2.2 분석 레이어 (4단계 체인)

1. **이벤트 추출** *(경량 모델)* — 수집 기사에서 거시 이벤트 추출 + 카테고리 분류 + 출처 링크 보존
2. **시나리오 대조** *(중급 모델)* — 시나리오 DB와 매칭 → 각 시나리오에 대해 `강화 / 약화 / 무관` 판정 + 근거
3. **파급 체인 생성** *(중급 모델)* — 사건 → 1차 파급 → 2차 파급 → 자산 영향(논리 트리). **각 화살표에 근거·신뢰도 부착**
4. **한국 자산 번역 + 시사점** *(중급 모델)* — 자산군별 관찰 포인트를 단기/중기/장기로, **특히 원화·코스피·한국 수출산업 관점으로 환산**

### 2.3 생성 레이어 (제약 생성)

- 입력: 분석 산출물 + **확정 팩트 목록**(숫자 경로 + tier 통과 인용만)
- 기사 템플릿: `헤드라인 / 리드 / 흐름 분석 / 시나리오 트리 / 한국 자산 시사점 / 시계열 관점(단·중·장) / 출처 목록 / 용어 해설`
- **톤 규칙**: 단정("사라"·"오른다") 금지 → 관점 제시("~할 가능성을 시사", "관찰 포인트는~"). 개별 종목·목표가·매수매도 금지(자산군 레벨만).
- **차별점 반영**: 매일 발행 시 "어제 대비 **무엇이 바뀌었나**"를 리드로 → 매일 같은 얘기 반복 방지.

### 2.4 검수 게이트 (3번째 방어선)

- 생성된 기사에서 **모든 수치·사실 주장**을 추출 → 팩트 목록·출처와 1:1 대조
- 미매칭 주장: **제거하거나 "미확인" 라벨**. 통과율을 메타데이터로 기록(품질 추적).
- *(경량 모델로 충분 — 생성과 다른 호출로 분리해야 자기검증 효과)*

### 2.5 전달 레이어

- **웹 대시보드**: 주기별 탭, 아카이브, 시나리오 트리 인터랙티브 뷰, 태그(자산·테마·시나리오) 검색, **시나리오 성적표**
- **PDF**: 월간·분기·연간 (기존 WeasyPrint 60.2 + pydyf 0.8.0 + Noto Sans CJK KR 파이프라인 재사용)
- **푸시(선택)**: 일간 시그널 → 슬랙/텔레그램

## 3. 시나리오 DB 스키마 (서비스의 핵심 자산)

```json
{
  "id": "scn-2026-cbdc-silver",
  "제목": "법정화폐 신뢰 약화 → CBDC 전환 → 실물 가치저장 수요(은)",
  "한줄요약": "화폐 시스템 불신이 커질수록 비(非)신뢰자산 수요가 늘어난다는 가설",
  "가설": "...",
  "상태": "강화중",                 // 활성 | 강화중 | 약화중 | 종료(적중) | 종료(빗나감)
  "신뢰도": 0.45,                   // 0~1, 갱신마다 조정
  "트리거_조건": [                  // 관찰되면 가설을 강화하는 신호
    "주요국 CBDC 파일럿 확대 발표",
    "실질금리 하락 + 금/은 동반 상승"
  ],
  "반증_조건": [                    // 관찰되면 가설을 약화/종료하는 신호
    "달러지수 추세적 강세 + 귀금속 동반 약세"
  ],
  "파급_체인": [
    {"단계": "사건", "내용": "화폐 신뢰 약화", "근거": "...", "출처": "..."},
    {"단계": "1차", "내용": "안전·대체자산 수요", "신뢰도": 0.6},
    {"단계": "2차", "내용": "금→은으로 확산(가격 레버리지)", "신뢰도": 0.4}
  ],
  "영향_자산": [
    {"자산": "은(silver)", "방향": "상승편향", "시계열": "중기", "신뢰도": 0.4},
    {"자산": "원/달러", "방향": "변동성확대", "시계열": "단기", "신뢰도": 0.3}
  ],
  "한국_자산_함의": "귀금속 ETF·관련주 변동성, 안전선호 시 원화 약세 압력",
  "이력_로그": [                    // ★ 적중 추적의 기반 — 1일차부터 기록
    {"날짜": "2026-06-11", "변화": "강화", "근거": "CBDC 파일럿 2개국 확대", "출처": "...", "브리핑id": "daily-2026-06-11"}
  ],
  "생성일": "2026-06-11",
  "최종갱신": "2026-06-11",
  "태그": ["통화정책", "귀금속", "지정학"]
}
```

설계 의도:
- **트리거/반증을 명시적으로** 둬서, 매 브리핑마다 "오늘 이벤트가 어느 시나리오의 트리거/반증을 건드렸나"를 기계적으로 대조 가능.
- **이력_로그**가 곧 "작년 이맘때 이 시나리오가 어떻게 전개됐는지"와 **적중 추적**의 원천. 기능(v0.4)은 나중이라도 **데이터는 1일차부터 쌓는다.**
- 상태에 `종료(빗나감)`을 둬서 **실패도 정직하게 기록** → 시나리오 성적표의 신뢰성.

## 4. 발행 주기별 콘텐츠 (유지) + 구동 방식

> **구동 결정(2026-06-11):** 토큰당 결제하는 별도 API가 아니라, **이미 내는 클로드 구독을
> 클로드 코드 자동 실행(headless)으로** 사용한다 → **추가 비용 0원, 품질은 Opus 그대로.**
> `web_search`도 클로드 코드 안에서 처리되어 별도 검색 비용이 없다. 따라서 v0.1·v0.2 초안에 있던
> "일간 비용 상한·토큰 가드레일·경량 모델 배분"은 **불필요**해져 폐기한다.

| 주기 | 이름(가칭) | 구동 | 분량 |
|------|-----------|------|------|
| 일간 | 데일리 시그널 | 클로드 코드 예약 실행(아침 1회) | 모바일 1화면 |
| 주간 | 위클리 플로우 | 클로드 코드 예약 실행(주 1회) | 중간 |
| 월간 | 먼슬리 테마 | 클로드 코드 예약 실행(월 1회) | 길게 |
| 분기 | 쿼터리 사이클 | 클로드 코드 예약 실행(분기 1회) | 길게 |
| 연간 | 애뉴얼 시나리오 | 클로드 코드 예약 실행(연 1회) | 리포트급 |

- 하위 주기는 상위 주기를 참조(연간=지도, 일간=현재 위치 업데이트). *유지*
- **유일한 가드레일은 구독 사용 한도**: 하루 한 번 도는 작은 작업이라 한도 대비 미미하나, 1차 구동 후
  실제 사용량을 작업로그에 한 줄 기록해 확인한다. (별도 결제·예산 역산 불필요)

## 5. MVP 범위 (재조정된 순서)

> v0.1의 "일간 브리핑부터"를 뒤집어, **참조 자산을 먼저** 만든 뒤 기사를 생성한다.

1. **시나리오 DB 스키마 확정 + 시드 10~15개** (위 3장 구조, 스크린샷류 거시 시나리오)
2. **구조화 데이터 수집** (핵심 시리즈 5~10개, `{값·시점·출처}` 저장)
3. **이벤트 수집** (web_search + tier 화이트리스트)
4. **분석 체인 4단계** (추출 → 시나리오 대조 → 파급 → 한국자산 시사점)
5. **제약 생성 + 검수 게이트** (팩트 목록 기반, 숫자 추가 금지, 대조 통과율 기록)
6. **마크다운/HTML 출력 + 로컬 수동 실행** → 결과 검증 → 이후 스케줄러 연결

### 이후 로드맵
- **v0.3**: 주간·월간 추가, PDF 발행, 웹 대시보드(아카이브 + 시나리오 트리 + 태그 검색)
- **v0.4**: 푸시 알림, **시나리오 성적표**(이력로그 기반 적중/빗나감 공개), RSS 확장

## 6. 리스크 & 원칙 (강화)

- **유사투자자문 규제 경계**: 개별 종목·목표가·매수매도 추천 **금지**. 자산군(asset class) 레벨 관점만. 모든 출력에 면책 문구.
  → 포지셔닝을 "학습·관찰 도구"로 명확히 하는 것이 법적 방어선이자 제품 정체성.
- **할루시네이션 방지**: 2장 "숫자/서사 분리" + 3중 게이트가 1차 방어. 모델 기억 기반 수치 생성 원천 차단.
- **출처 신뢰 등급(tier)**:
  - Tier 1: 공식기관(중앙은행·통계청·거래소·정부)
  - Tier 2: 주요 언론(연합·한경·매경·Bloomberg·Reuters)
  - Tier 3: 기타 — **"미확인" 라벨 필수, 수치 인용 불가**
  - SNS·익명·블로그: **차단**
- **반복·진부화 방지**: 일간은 "어제 대비 변화"를 핵심으로.
- **품질 측정**: 검수 게이트 통과율, 시나리오 적중률을 메타데이터로 누적 → 서비스가 실제로 "맞는지" 자가 점검.

## 7. 기술 스택 (유지 + 보강)

| 구성 | 제안 | 비고 |
|------|------|------|
| 백엔드 | Python | 기존 자동화 스택 동일 |
| AI | **클로드 코드 자동 실행(headless), 기존 구독 사용** (web_search 포함) | 분석 체인 + 제약 생성 + 검수. **토큰당 API 결제 없음** |
| 구조화 데이터 | FRED API, 한국은행 ECOS API, yfinance | **MVP 핵심으로 승격** |
| 스케줄러 | cron / 클로드 코드 예약 실행 | 일간 07:00, 주간 월, 월간 1일 |
| 저장소 | SQLite (시나리오 DB·아카이브·이력로그) | JSON보다 이력 누적·조회에 유리 |
| 프론트 | 정적 사이트 또는 Streamlit | MVP 경량 |
| PDF | WeasyPrint 60.2 + pydyf 0.8.0 + Noto Sans CJK KR | 검증된 조합 재사용 |

> 저장소를 v0.1의 "SQLite 또는 JSON"에서 **SQLite 우선**으로: 이력로그·적중추적·태그검색이 핵심이라 관계형이 유리.
